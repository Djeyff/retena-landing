Política de Seguridad

Última actualización: 13 de marzo de 2026

En Retena, la seguridad de tu información es nuestra máxima prioridad. Esta política describe nuestras medidas de seguridad y procedimientos para proteger tus datos de voz y comunicaciones grupales.

🔒 Arquitectura de Seguridad

Cifrado de Extremo a Extremo

Todas las transcripciones están cifradas en reposo con AES-256. Los archivos de audio se procesan y eliminan inmediatamente; solo las transcripciones de texto se almacenan de forma segura.

  • Datos en tránsito: TLS 1.3 para todas las comunicaciones
  • Datos en reposo: Cifrado AES-256 con rotación de claves
  • Claves de acceso: Gestión segura con AWS KMS
  • Audio temporal: Se elimina automáticamente tras la transcripción

Acceso y Autenticación

  • Autenticación de dos factores (2FA) obligatoria para administradores
  • Tokens de sesión con caducidad automática
  • Registro completo de accesos y actividades
  • Principio de menor privilegio para todos los roles

🛡️ Protección de Datos

Retención Limitada

Implementamos políticas estrictas de retención de datos:

  • Audio: Eliminado inmediatamente después del procesamiento (máximo 30 segundos)
  • Transcripciones: Retenidas según configuración del cliente (30-365 días)
  • Metadatos: Almacenados el tiempo mínimo necesario para funcionalidad
  • Logs de acceso: 90 días máximo para auditoría y seguridad

Anonimización y Privacidad

Retena solo procesa los grupos donde es explícitamente añadida. No tenemos acceso a conversaciones privadas o grupos no autorizados.

  • Separación completa entre diferentes organizaciones
  • No se comparten datos entre clientes
  • Opción de eliminar datos bajo solicitud (derecho al olvido)
  • Configuración granular de permisos por grupo

🏗️ Infraestructura Segura

Alojamiento y Cumplimiento

  • Proveedores: AWS y Google Cloud (certificados SOC 2 Type II)
  • Ubicación: Centros de datos en UE para clientes europeos
  • Cumplimiento: GDPR, CCPA, SOC 2, ISO 27001
  • Backups: Cifrados y distribuidos geográficamente

Monitoreo y Respuesta a Incidentes

  • Monitoreo 24/7 de sistemas y seguridad
  • Alertas automáticas para actividad sospechosa
  • Plan de respuesta a incidentes con tiempos definidos
  • Auditorías de seguridad trimestrales por terceros

👥 Acceso de Empleados

Control Estricto

El acceso a datos de clientes está estrictamente limitado y registrado. Solo personal autorizado con necesidad comercial legítima puede acceder a transcripciones para soporte técnico.

  • Verificación de antecedentes para todo personal técnico
  • Acceso temporal y registrado para resolución de problemas
  • Prohibición absoluta de acceso no autorizado
  • Auditorías regulares de permisos y accesos

🔄 Actualizaciones de Seguridad

Mejora Continua

  • Actualizaciones automáticas de seguridad en sistemas críticos
  • Testing de penetración semestral
  • Revisión continua de políticas de seguridad
  • Capacitación regular del equipo en mejores prácticas

Disclosure Responsable

  • Resolution: Parches críticos implementados en 72 horas máximo
  • Recompensa: Programa de bug bounty para investigadores

✅ Certificaciones y Estándares

Retena cumple con los más altos estándares de seguridad de la industria:

  • SOC 2 Type II: Controles de seguridad, disponibilidad y confidencialidad
  • ISO 27001: Sistema de gestión de seguridad de la información
  • GDPR: Regulación General de Protección de Datos (UE)
  • CCPA: Ley de Privacidad del Consumidor de California
  • PIPEDA: Ley de Protección de Información Personal (Canadá)
